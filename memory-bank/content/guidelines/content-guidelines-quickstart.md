## Most Used Patterns

### Hero Headlines (Pick One Formula)

**Formula 1: Outcome + Method**
```
[Desired Outcome] with [Your Unique Method/Platform]
Example: "Transform Your IT Infrastructure with One SLA Simplicity"
```

**Formula 2: Imperative + Benefit**
```
[Action Verb] [What] to [Business Outcome]
Example: "Migrate Mission-Critical Workloads with Zero Downtime"
```

**Formula 3: Challenge + Solution**
```
[Industry Challenge]? [Your Solution]
Example: "Managing Multiple Vendors? Simplify with One SLA"
```

### CTA Buttons (Always Be Specific)

**Good:**
- Schedule Your Migration Assessment
- Request CloudEdge Demo
- Download SOC Services Overview
- Speak with a Security Specialist

**Bad:**
- Learn More
- Click Here
- Submit
- Find Out More

## Terminology Quick Reference

### Always Capitalize
- CloudEdge Portal (then "CloudEdge")
- Posta Portal (then "Posta")
- One SLA
- Zero Trust Architecture
- SOC, SIEM, SOAR, XDR, EDR (spell out first mention)

### Always Hyphenate
- White-label (when adjective)
- Multi-tenant
- Multi-cloud
- Zero-downtime (when adjective)
- Mission-critical (when adjective)

### Common Mistakes to Avoid
- ❌ "white label platform" → ✅ "white-label platform"
- ❌ "zero downtime migration" → ✅ "zero-downtime migration"
- ❌ "multi cloud" → ✅ "multi-cloud"
- ❌ "cloudedge portal" → ✅ "CloudEdge Portal"
- ❌ "Best in market" → ✅ "Trusted by 500+ enterprises" (specific proof)

## Voice & Tone: The Roaya IT Way

### DO Write Like This
✅ "Roaya IT delivers 24/7 SOC monitoring with AI-powered threat detection"
✅ "Reduce costs by 40% while improving security"
✅ "Migrate 500+ servers with zero downtime"
✅ "Schedule Your Infrastructure Assessment"

### DON'T Write Like This
❌ "Roaya IT leverages synergistic paradigms for next-gen solutions"
❌ "Significant cost savings can be achieved"
❌ "Migration services are provided"
❌ "Learn More"

### Writing Rules
1. **Active voice:** "Roaya IT delivers" not "Services are delivered"
2. **Front-load benefits:** Start with outcome, follow with feature
3. **Be specific:** "40% cost reduction" beats "significant savings"
4. **Short paragraphs:** 2-3 sentences maximum
5. **Clear CTAs:** Tell users exactly what to do next

## SEO Essentials

### Title Tag Template
```
[Primary Keyword] | [Differentiator] | Roaya IT
Example: CloudEdge Portal | White-Label Cloud Management | Roaya IT
Length: 50-60 characters
```

### Meta Description Template
```
[What you deliver] for [target audience]. [Benefit 1], [Benefit 2], [Benefit 3]. [CTA].
Example: White-label cloud management for service providers. Unlimited customization, multi-tenant architecture, seamless integrations. Request demo.
Length: 150-160 characters
```

### Heading Hierarchy
- **H1:** One per page, primary keyword, 5-12 words
- **H2:** Major sections, secondary keywords, 3-8 words
- **H3:** Subsections, long-tail keywords, 2-6 words
- **H4+:** Rare, only for deeply nested content

## Microcopy: Common Scenarios

### Form Validation
```
Email required: "Email address is required"
Invalid email: "Please enter a valid email address"
Password weak: "Password must include uppercase, lowercase, and numbers"
Success: "Thank you! We'll contact you within 24 hours."
Error: "Something went wrong. Try again or contact support@wpsys.com"
```

### Button States
```
Default: "Request Demo"
Loading: "Submitting..."
Success: "Request Submitted ✓"
Error: "Submission Failed. Try Again."
```

### Empty States
```
No results: "No results found for '[term]'. Try different keywords or browse our services below."
No data: "No data available yet. Check back later or contact support."
```

## Common Questions

### Q: When do I capitalize "white-label"?
**A:** Never capitalize unless it starts a sentence. Always hyphenate when used as adjective: "white-label platform"

### Q: How do I reference CloudEdge after first mention?
**A:** First mention: "CloudEdge Portal" → Subsequent: "CloudEdge"

### Q: What's the difference between "cloud provider" and "service provider"?
**A:**
- **Cloud service provider (CSP):** Specifically offers cloud computing resources
- **Service provider:** Broader term (includes hosting, MSPs, etc.)
- **Hosting provider:** Focuses on web/server hosting

### Q: Should I spell out acronyms?
**A:** Yes, on first mention in body copy: "Security Operations Center (SOC)" → then "SOC"
Exception: Very common terms (API, CRM, VPN) don't need spelling out in technical content

### Q: What tone for technical vs business audience?
**A:** Roaya IT targets IT Directors and CTOs—balance technical credibility with business outcomes. Use technical terms correctly but focus on ROI, efficiency, and risk reduction.

## Quick Checklist Before Publishing

**Content:**
- [ ] Headline uses proven formula (Outcome + Method / Imperative + Benefit)
- [ ] First paragraph includes primary keyword naturally
- [ ] Benefits front-loaded before features
- [ ] Metrics and proof points included (99.99% uptime, 40% cost reduction)
- [ ] Paragraphs are 2-3 sentences maximum
- [ ] Bullet points start with verbs

**SEO:**
- [ ] Title tag 50-60 characters with primary keyword
- [ ] Meta description 150-160 characters with CTA
- [ ] H1 includes primary keyword (5-12 words)
- [ ] H2/H3 structure is logical and scannable
- [ ] Internal links use keyword-rich anchor text

**Voice & Tone:**
- [ ] Active voice used throughout
- [ ] Specific claims backed by proof points
- [ ] Technical terms used correctly
- [ ] No buzzwords without substance
- [ ] CTAs are action-oriented and specific

**Terminology:**
- [ ] CloudEdge Portal, Posta Portal capitalized
- [ ] One SLA capitalized
- [ ] White-label, multi-tenant, multi-cloud hyphenated
- [ ] Acronyms spelled out on first mention (SOC, SIEM, etc.)

**Microcopy:**
- [ ] Form validation messages are helpful
- [ ] Button states designed (loading, success, error)
- [ ] Error messages provide solutions
- [ ] Empty states guide next action
